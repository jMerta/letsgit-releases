# Security Policy

## Reporting a vulnerability

If you discover a security vulnerability in LetsGit, **please report it privately**. Do not open a public GitHub issue.

**Email:** [support@letsgit.app](mailto:support@letsgit.app)

Include as much detail as possible:
- Description of the vulnerability
- Steps to reproduce
- Affected version(s)
- Potential impact

## Response

I aim to acknowledge reports within 48 hours and provide a fix or mitigation plan within 7 days for confirmed issues.

## Scope

This policy covers:
- The LetsGit desktop application
- The auto-update mechanism
- The local data plane (HTTP server on localhost)

Out of scope:
- The marketing website (letsgit.app)
- Third-party dependencies (report those to the upstream project)
